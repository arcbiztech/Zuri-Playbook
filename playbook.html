What This Agent Does

This is a production grade AI agent built to automate an entire job search pipeline. From discovery to a tailored application draft ready for candidate review, with zero manual searching required.

It runs on a schedule, finds real job listings from targeted sources, scores each one across seven dimensions aligned to a specific candidate profile, and generates a tailored draft resume and cover letter for high scoring matches. These drafts go to the candidate for review, quality check, and refinement before any application is submitted.

This playbook documents the agent's architecture, logic, decision rules, and team touchpoints, written so any cross functional team member can understand, configure, and build on it.

Human in the Loop

The AI surfaces and scores opportunities. Every application decision stays with the human. The agent advises, it does not decide.

Config Driven, Not Hard Coded

Every parameter (keywords, salary floor, scoring weights, job boards) lives in an Airtable Config table. Anyone on the team can update the system by editing a single cell. No automation editing required.

Modular by Design

Each scenario has exactly one job. A failure in sourcing does not break scoring. A scoring delay does not block document generation. Built the way professional software is built.

Why This Approach

Why an Agent Based Approach Matters for Teams

Most organizations still run high stakes, repeatable processes manually, recruitment, lead qualification, onboarding, reporting. This agent shows what becomes possible when those processes are designed with AI as a real workflow partner.

Consistency at Scale

Every job gets evaluated against the same weighted criteria every single time. No variation based on who ran the search or what day it was. The system applies the same thoughtful judgment at scale that a skilled human would apply to a handful of listings.

Time Returned to High Value Work

Manual searching, deduplication, scoring, and document tailoring can eat hours every week. This agent collapses all of that into a Monday morning review of pre scored, pre drafted applications, so the human can focus on decisions only a human can make.

Intelligence That Compounds

Every run adds to the outcome dataset. Over time the team learns which sources produce real interviews, which score ranges convert, and which keywords land. The system gets smarter the longer it runs, no developer required to adjust it.

The bigger picture: This pipeline is a proof of concept for any team running repetitive, judgment based workflows. The same architecture, config driven parameters, modular scenarios, AI scoring, human review, can be applied to customer onboarding, content workflows, sales qualification, vendor evaluation, and more. The agent does not replace the team. It removes the mechanical work so the team can focus on what actually requires them.

System Architecture

Four Scenarios. One Closed Loop.

Each scenario has one job and fails independently. A broken data source does not take down the scoring logic. All configuration lives in Airtable, change any parameter by editing one cell, no automation editing required.

01

Scenario 1: Controller

Reads all parameters from the Config table, splits keywords into an array, queries Tavily for job listings per keyword, and sends each result to Scenario 2. Fires a weekly completion email when the run finishes.

02

Applies automated filters and deduplicates each job against the existing Jobs table. Forwards only new, qualified listings to Scenario 3 for scoring.

03

Claude researches the company and scores 7 dimensions. Make calculates the weighted formula, not the AI. A 4 way router stores each job under the correct score tier in Airtable.

04

Fetches high scoring jobs, reads the correct base documents from Google Drive, sends them to Claude for targeted injection, creates a customized output document, and emails the link.

Filters run before anything else

The moment a job arrives, it hits a filter gate that evaluates the job title, content, and source URL simultaneously. All conditions must pass, any single failure stops the job right there. Only listings that clear every filter move forward to the deduplication check. This protects the AI scoring step from wasting time and tokens on jobs that were never a real fit.

Every job URL is checked against Airtable

After passing the filter gate, each job's URL is searched against the full Jobs table. If a match is found, the job is skipped entirely. If no match is found, it proceeds to Scenario 3. The same listing can never be scored twice across weekly runs, keeping the database clean and the scoring costs down.

Only qualified, new jobs reach the AI

The final webhook only fires when a job has passed both the filter gate and the deduplication check. The complete payload, job data plus all scoring weights from Config, is forwarded to Scenario 3. Everything that arrives at the scoring step has already earned its place there.

Filter Logic

Multiple filter conditions applied automatically to each incoming job
Filters evaluate the job title, content, and source URL simultaneously
All filter conditions must pass. Any single failure stops the job.
Specific filter criteria are configurable and not publicly disclosed

Deduplication Logic

Every incoming job URL is checked against the full Jobs table
Match found: job is skipped. No match found: job proceeds to scoring.
The same listing is never scored twice across multiple weekly runs
Deduplication happens before the Claude API call, saving tokens and cost.

Claude researches the company before scoring

For every qualified job that arrives, Claude uses web search to look up the company before assigning a single score. It checks Glassdoor ratings, culture signals, mission language, salary transparency, and founding details, then scores all 7 dimensions from 1 to 10 in 0.5 increments only. The result comes back as a strict JSON object with scores, notes, resume and cover letter suggestions, and any flags.

Make calculates the formula, not the AI

This is a deliberate architectural choice. AI math drifts and cannot be audited, so Claude returns the raw dimension scores and Make runs the weighted formula: CF times weight plus SF times weight plus SK times weight and so on across all 7 dimensions. The result is a true floating point decimal, consistent and verifiable every time.

A 4 way router stores jobs by score tier

Once the score is calculated, a router confirms the URL exists and the score meets the minimum threshold from Config. Then it routes the job to one of four parallel Airtable branches, Dream Role, Must Apply, Strong Fit, or Monitor. Each branch creates one complete Airtable record with all 22 fields populated and Status defaulted to Ready to Apply.

Scoring System

7 Dimension Weighted Formula

Claude scores each dimension from 1 to 10 in 0.5 increments only. Whole numbers are not permitted. Make calculates the final weighted score. All weights sum to 100% and are configurable in the Airtable Config table.

Document Selection

Why There Are Multiple Resume and Cover Letter Versions

The Philosophy Behind Multiple Versions

Every resume and cover letter in this system was written entirely by hand, not generated by AI,, not templated, and not interchangeable. Each version reflects a specific industry context, a different set of values the employer is likely to hold, and a distinct part of my professional story that resonates most with that type of organization.

They are not changed by the automation. They do not get rewritten. They are preserved in full because the human voice, the personal narrative, and the philosophical framing are the most important things a document can carry, and those are the first things lost when AI takes over the writing.

Because every job that reaches document generation has already scored very high, the keywords being injected are inherently relevant. High scoring roles are already aligned to my skills and experience. Injecting those keywords into existing, genuinely written content is a targeted, honest optimization, not a fabrication.

Wellness and EdTech Resume

When: Wellness, mindfulness, mental health, EdTech, language learning, nonprofits, NGOs, social impact, global health, community organizations

Why Make Calculates the Score

AI math drifts. The same inputs can return different outputs.
Make runs the formula deterministically every single time
Results are verifiable, auditable, and not subject to AI variation
Weights are configurable in Airtable without touching the automation

Why 0.5 Increment Scoring

Whole numbers create too many ties in the final weighted score
Half point precision gives Claude meaningful differentiation
Results in cleaner tier separation and more accurate routing
The system prompt explicitly instructs Claude to never use whole numbers

Claude injects. It does not rewrite.

Claude receives the full job title, company name, job description, base resume text, and base cover letter text. The instruction is clear, preserve both documents in full and inject only what belongs. Company name, job title, and relevant keywords go in where they fit naturally. If a keyword does not fit cleanly, it gets skipped entirely. The voice and narrative structure stay exactly as written. The result is a draft that gives the candidate a strong, informed starting point, not a finished document.

A finished document is created and linked back

The URL is written back to the Resume Link field in Airtable and the Status updates to Draft Ready. A final email delivers every document link in one place. This is a structured draft, not a finished application. The candidate reviews the draft, checks quality, and uses the research and company context already gathered to add the kind of thoughtful, refined detail that only they can bring.

The Injection Standard: What Claude Does and Does Not Change

The original resume and cover letter are preserved in full. Both documents are saved exactly as written before any changes are made. What Claude does is targeted and minimal, inject the company name, the specific job title, and relevant keywords where they fit naturally into the existing text, and optionally add one or two factual details about the company that strengthen what is already being said.

What Claude Injects Into the Resume

Job title updated in the header to align with the target role
Summary, relevant keywords woven naturally into existing sentences
Up to 3 bullet points, only where a keyword fits without forcing it
If a keyword does not fit naturally it is skipped entirely

What Claude Injects Into the Cover Letter

Company name updated throughout wherever the hiring team is referenced
Job title updated wherever the role is referenced in the text
Relevant keywords injected naturally into existing sentences only
1 to 2 company facts, only if they strengthen something already being said

The candidate's own words are the most important element of every document. The philosophical framing, personal stories, voice, and narrative structure are never touched. What arrives in the output document should feel indistinguishable from something written personally for that company, because it was. The automation handles the mechanical work. What arrives in the draft is a strong, informed starting point. The candidate takes it from there, adding the depth and personal insight that no automation can replicate. This is a draft. The human elevates it.

Claude Output Format

ATS KEYWORDS, top 10 keywords extracted from the job description
RESUME, full original resume with minimal surgical injections only
COVER LETTER, full original cover letter with targeted updates only
SUMMARY, top 3 alignment points for the reviewer's quick scan
